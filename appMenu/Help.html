Overview

appMenu lets you organize your apps into folders and access them from the menu bar and the Dock menu.

Launching Apps

  • Menu bar: click the grid icon, then click any app.
  • Dock: right-click the appMenu Dock icon and choose an app.

Opening Settings

  • Main window: click Open Settings.
  • Menu bar or Dock menu: choose Open Settings….

Settings Window

The Settings window shows a tree of folders and apps. Create folders, add apps, and reorder items. Changes are saved automatically.

Creating and Managing Folders

  • Add at root: toolbar → Add Folder.
  • Add inside a folder: folder row → + → Add Folder.
  • Rename: click the name and edit inline.
  • Delete: select and click Delete or the trash button.

Adding Applications

  1. Root level
    • Toolbar → Add Apps: pick apps via an open panel.
    • Toolbar → Import from /Applications…: open a searchable checklist of apps.
  2. Specific folder
    • Folder row → + → Add Apps…: pick apps; duplicates in that folder are skipped.
    • Folder row → + → Choose from /Applications…: open the checklist targeted to that folder; duplicates are skipped.

Search and Checklist Import

  • Filter by name or filename (e.g., “Microsoft”, “Adobe”).
  • Checkbox list for easy multi-select; use Select All/Select None on the filtered list.
  • Duplicates already present in the target folder (or root) are not added.

Reordering and Drag-and-Drop

  • Drag items to reorder or move into folders.
  • Dropping onto a folder appends it as a child.
  • Cycles and no-op moves are prevented.

Refreshing Menus

  • Menus update automatically as you edit.
  • Manual refresh: main window → Refresh Menus, or menu bar/Dock menu → Refresh.

Quitting the App

  • Menu bar or Dock menu → Quit, or quit from the app menu.

Persistence

Your setup is saved in Application Support and restored at launch.

Keyboard Shortcuts

  • Enter/Return commits inline edits.
  • Delete removes the selected item.
  • In the import sheet: Return = Add Selected, Escape = Cancel.

Troubleshooting

  • If an app isn’t found by import, use Add Apps… to pick it from its location.
  • If you see apparent duplicates, they may be different copies on disk; duplicates within the same folder are skipped when adding.
  • On older macOS versions, a fallback Settings window is used.

Privacy and Permissions

appMenu reads your local filesystem to find and open apps. It stores configuration locally and requires no network access.

Tips

  • Group by vendor or role (e.g., “Microsoft”, “Adobe”, “Browsers”, “Editors”).
  • Use search + Select All in the import sheet to add many related apps at once.
